Demarrage

Guide d'installation et d'utilisation du NXO Design System dans vos projets.

Installation

Ajoutez les fichiers CSS du Design System a votre projet.

<!-- 1. Police Montserrat -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&display=swap" rel="stylesheet">

<!-- 2. Font Awesome pour les icones -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">

<!-- 3. Design System CSS -->
<link rel="stylesheet" href="tokens/tokens.css">
<link rel="stylesheet" href="components/components.css">

Structure des fichiers

nxo-design-system/
├── tokens/
│   └── tokens.css       # Variables CSS (couleurs, espacements, etc.)
├── components/
│   └── components.css   # Styles des composants
├── assets/
│   ├── logos/              # Logos NXO / Fayat
│   └── icons/              # Icones SVG
└── docs/
    └── *.html              # Documentation

Utilisation basique

Exemple de page HTML utilisant le Design System.

<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Mon Application NXO</title>

  <!-- Design System -->
  <link href="https://fonts.googleapis.com/css2?family=Montserrat:wght@300;400;500;600;700&display=swap" rel="stylesheet">
  <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">
  <link rel="stylesheet" href="tokens/tokens.css">
  <link rel="stylesheet" href="components/components.css">
</head>
<body>

  <header class="nxo-header">
    <img src="assets/logos/logo.png" alt="NXO">
    <h1>Mon Application</h1>
  </header>

  <main style="padding: var(--nxo-spacing-xl);">
    <div class="nxo-card">
      <h2 class="nxo-h3">Bienvenue</h2>
      <p class="nxo-text">Votre contenu ici...</p>
      <button class="nxo-btn nxo-btn-primary">Action</button>
    </div>
  </main>

</body>
</html>

Utilisation des variables CSS

Utilisez les tokens pour personnaliser vos composants.

/* Utiliser les couleurs */
.mon-element {
  background: var(--fayat-blue);
  color: var(--fayat-yellow);
}

/* Utiliser les espacements */
.mon-conteneur {
  padding: var(--nxo-spacing-lg);
  gap: var(--nxo-spacing-md);
}

/* Utiliser la typographie */
.mon-titre {
  font-size: var(--nxo-font-size-2xl);
  font-weight: var(--nxo-font-weight-bold);
}

/* Utiliser les radius */
.ma-carte {
  border-radius: var(--nxo-radius-lg);
}

Mode sombre

Le Design System supporte le mode sombre nativement.

<!-- Ajouter ce script dans <head> pour eviter le flash -->
<script>
  (function() {
    var theme = localStorage.getItem('nxo-theme');
    if (theme === 'dark' || (!theme && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
      document.documentElement.setAttribute('data-theme', 'dark');
    }
  })();
</script>

<!-- Bouton pour basculer le theme -->
<button onclick="toggleTheme()">Changer de theme</button>

<script>
function toggleTheme() {
  const html = document.documentElement;
  const isDark = html.getAttribute('data-theme') === 'dark';
  html.setAttribute('data-theme', isDark ? 'light' : 'dark');
  localStorage.setItem('nxo-theme', isDark ? 'light' : 'dark');
}
</script>

Bonnes pratiques

A faire

  • Utiliser les variables CSS du Design System
  • Respecter les espacements definis
  • Utiliser les composants existants
  • Tester en mode clair ET sombre

A eviter

  • Coder des valeurs en dur (couleurs, tailles)
  • Modifier les tokens globalement
  • Ignorer l'accessibilite
  • Creer des composants non documentes
