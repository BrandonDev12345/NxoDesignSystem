Responsive

Tous les composants du Design System sont responsive nativement. Deux breakpoints principaux : 768px (tablette/mobile) et 480px (petit mobile).

Tables responsive

Pour activer le scroll horizontal sur les tables en mobile, encapsulez la table dans un conteneur .nxo-table-responsive : 

<div class="nxo-table-responsive">
  <table class="nxo-table">
    ...
  </table>
</div>

Sidebar responsive

La sidebar necessite un bouton hamburger, un overlay et un peu de JavaScript. Voir la documentation Sidebar pour le code complet. 

<!-- Ajouter dans votre HTML -->
<button class="nxo-sidebar-toggle" id="sidebarToggle">
  <i class="fa-solid fa-bars"></i>
</button>
<div class="nxo-sidebar-overlay" id="sidebarOverlay"></div>
<aside class="nxo-sidebar" id="sidebar">...</aside>

<script>
  const toggle = document.getElementById('sidebarToggle');
  const sidebar = document.getElementById('sidebar');
  const overlay = document.getElementById('sidebarOverlay');
  toggle.addEventListener('click', () => {
    sidebar.classList.toggle('open');
    overlay.classList.toggle('open');
  });
  overlay.addEventListener('click', () => {
    sidebar.classList.remove('open');
    overlay.classList.remove('open');
  });
</script>

Bonnes pratiques

Faire
Toujours ajouter <meta name="viewport"> dans le <head>. Utiliser .nxo-table-responsive autour des tables larges. Tester sur un vrai mobile.
Eviter
Largeurs fixes en pixels sur les conteneurs. Desactiver le zoom utilisateur. Utiliser des font-size < 16px sur les inputs (zoom iOS).
