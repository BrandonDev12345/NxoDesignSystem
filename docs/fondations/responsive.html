Responsive

Tous les composants du Design System sont responsive nativement. Deux breakpoints principaux : 768px (tablette/mobile) et 480px (petit mobile). Aucun JavaScript n'est necessaire (sauf pour la sidebar).

Breakpoints

BreakpointCibleComposants concernes
1440px Grand desktop Header (zones reduites)
1024px Desktop / tablette landscape Header (burger visible, icones reduites)
768px Tablette / mobile Tous les composants (typo, cards, modals, tabs, sidebar, footer, stepper, nav, accordion, alerts, pagination, breadcrumb, dropzone, lists, toasts, btn-group)
480px Petit mobile Header (logo mobile), typo, cards, inputs (44px tactile), btn-group, pagination, empty state, timeline

Tables responsive

Les tables larges debordent sur mobile. Encapsulez-les dans .nxo-table-responsive pour activer le scroll horizontal :

<div class="nxo-table-responsive"> <table class="nxo-table"> <thead>...</thead> <tbody>...</tbody> </table> </div>

Sidebar responsive

La sidebar est le seul composant qui necessite du JavaScript pour le responsive. En dessous de 768px, elle se cache et un bouton hamburger apparait.

Voir la documentation Sidebar pour les exemples React, Angular, et PHP/Twig.
<!-- 1. Bouton hamburger (visible auto en mobile) --> <button class="nxo-sidebar-toggle" id="sidebarToggle"> <i class="fa-solid fa-bars"></i> </button> <!-- 2. Overlay sombre --> <div class="nxo-sidebar-overlay" id="sidebarOverlay"></div> <!-- 3. Sidebar classique --> <aside class="nxo-sidebar" id="sidebar"> ... </aside> <!-- 4. JavaScript --> <script> const toggle = document.getElementById('sidebarToggle'); const sidebar = document.getElementById('sidebar'); const overlay = document.getElementById('sidebarOverlay'); toggle.addEventListener('click', () => { sidebar.classList.toggle('open'); overlay.classList.toggle('open'); }); overlay.addEventListener('click', () => { sidebar.classList.remove('open'); overlay.classList.remove('open'); }); </script>

Bonnes pratiques

A faire
  • Toujours inclure <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Utiliser .nxo-table-responsive autour des tables de plus de 3-4 colonnes
  • Utiliser les classes .nxo-btn-group pour grouper les boutons (s'empilent automatiquement)
  • Ajouter .nxo-pagination-ellipsis sur les items "..." de la pagination
  • Tester sur un vrai mobile ou avec les DevTools en mode responsive
A eviter
  • Largeurs fixes en pixels sur les conteneurs principaux
  • Desactiver le zoom utilisateur (user-scalable=no)
  • Utiliser des font-size < 16px sur les <input> (provoque un zoom auto sur iOS)
  • Forcer overflow: hidden sur le body (casse le scroll mobile)
